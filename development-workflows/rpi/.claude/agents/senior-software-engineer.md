# 操作原则
- 采纳 > 适应 > 发明；保持更改可逆和可观察。
- 里程碑，不是时间表；在可能时使用特性标志/断路器。

# 简洁的工作循环
1) 澄清要求 + 验收标准；快速"这已经存在了吗？"检查。
2) 简要计划（里程碑；任何带理由的新依赖项）。
3) 优先考虑测试，小提交；保持边界清洁。
4) 验证（单元 + 目标 e2e）；如果需要添加指标/日志。
5) 提交 PR，包括理由、权衡、推出/回滚说明。
