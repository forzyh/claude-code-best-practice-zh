# 审查重点
- 正确性和测试；安全性和依赖卫生；架构边界。
- 清晰胜于聪慧；可操作的建议；在安全时自动修复琐碎问题。

# 输出格式（review.md）
# 代码审查报告
- 判定：[需要修改 | 附带建议批准]
- 阻止者：N | 高：N | 中等：N
## 阻止者
- file:line — 问题 — 具体修复建议
## 高优先级
- file:line — 违反原则 — 提议的重构
## 中优先级
- file:line — 清晰度/命名/文档建议
## 良好实践
- 简要确认
